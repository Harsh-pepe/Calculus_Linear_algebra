Act One

Data Tables as Vectors and Matrices

Every dataset you will ever encounter — a spreadsheet of patient vitals, a table of sensor readings from an IoT deployment, a million rows of e-commerce transactions — is, at its most fundamental level, a matrix. Each row is a vector in some high-dimensional space. This is not a poetic metaphor. It is a precise mathematical statement, and treating it seriously unlocks an enormous amount of power.

The Data Matrix

Suppose you are building a model to predict house prices. Your dataset has n = 10,000 houses (observations) and d = 12 features: area, number of bedrooms, age, location coordinates, and so on. Mathematically, this is a matrix X ∈ ℝ^{n×d} — a rectangle of real numbers with 10,000 rows and 12 columns. Each row x_i ∈ ℝ^d is a data point: a vector in 12-dimensional space representing one house.

X = [ x₁ | x₂ | ... | xₙ ]ᵀ ← n×d data matrix

x_i = [ area, bedrooms, age, lat, lon, ... ] ← single data point vector

‖x_i - x_j‖ = Euclidean distance between two houses in feature space

Operations That Yield Insight

1. The Dot Product as Similarity

The dot product a · b = ‖a‖ ‖b‖ cos θ measures how aligned two vectors are. In a recommendation system (think Netflix, Spotify, Amazon), each user is a vector of ratings and each item is a vector of attributes. The dot product between a user vector and an item vector is a similarity score. This is collaborative filtering — one of the most commercially valuable algorithms ever deployed — and it is nothing more than a repeated dot product over a large matrix.

2. Matrix Multiplication as Batch Transformation

When you multiply the data matrix X by a weight matrix W, you are simultaneously transforming every single data point. In a neural network, every layer is a matrix multiplication followed by a non-linear function. The entire forward pass of GPT-4 — a model with trillions of floating point values — is an enormously complex chain of matrix multiplications. The efficiency of modern hardware (GPUs, TPUs) is fundamentally designed around doing matrix multiplications extremely fast.

3. The Norm as a Measure of Magnitude

The L2 norm ‖x‖ = √(x₁² + x₂² + ... + xd²) tells you the "size" or "magnitude" of a data vector. In machine learning, norms appear everywhere: in the loss function (Mean Squared Error is an L2 norm), in regularisation (Ridge regression penalises the L2 norm of weights, Lasso penalises the L1 norm), and in distance-based algorithms like K-Nearest Neighbours.

vectors_and_similarity.py 
import numpy as np

# Simulate a tiny data matrix: 5 houses, 3 features
# [area_sqft, num_bedrooms, age_years]
X = np.array([
    [1200, 3, 10],
    [1800, 4, 5],
    [800,  2, 20],
    [2200, 5, 2],
    [1500, 3, 15],
], dtype=float)

# Normalise rows to unit vectors (direction matters, not magnitude)
norms = np.linalg.norm(X, axis=1, keepdims=True)
X_norm = X / norms

# Cosine similarity matrix — dot product on unit vectors
sim_matrix = X_norm @ X_norm.T
print("Cosine Similarity Matrix:")
print(np.round(sim_matrix, 3))

# Most similar house to house 0?
row0 = sim_matrix[0].copy()
row0[0] = -1   # exclude self
print(f"House 0's nearest neighbour: House {np.argmax(row0)}")

Act Two

Linear Transformations and Their Impact on Data

A linear transformation is a function T: ℝⁿ → ℝᵐ that maps every vector in one space to a vector in another, while preserving the operations of addition and scalar multiplication. Every such transformation can be represented as a matrix multiplication: T(x) = Ax. But what does this do to data — geometrically, visually, practically?

The Four Fundamental Actions

Scaling

Multiplying by a diagonal matrix stretches or squashes the data along axes. Standardising features before training an ML model (z = (x − μ) / σ) is a linear transformation — it scales the feature space so all dimensions have unit variance, preventing large-magnitude features from dominating the model.

Rotation

An orthogonal matrix Q (where QQᵀ = I) rotates the data cloud in space without distorting distances. PCA uses a rotation to align the data with its axes of maximum variance — more on this in Section 10.

Projection

Mapping a high-dimensional vector to a lower-dimensional subspace is a projection. Every dimensionality reduction technique — PCA, LDA, autoencoders — is, at its core, a carefully chosen projection that preserves the most important structure.

Shearing

Some transformations tilt the data cloud, changing the relationships between features. Understanding these shears helps interpret why some feature engineering steps distort or improve model performance.

Neural Networks as Chains of Transformations

A fully connected neural network is literally a sequence of linear transformations interleaved with non-linear activation functions:

h₁ = σ(W₁x + b₁) ← Layer 1: linear transform + activation
h₂ = σ(W₂h₁ + b₂) ← Layer 2: transform the transformed
ŷ = softmax(W₃h₂ + b₃) ← Output: class probabilities

Each weight matrix Wₖ is performing a linear transformation. The non-linearity (ReLU, sigmoid, tanh) between layers is what makes the network capable of learning curved decision boundaries. Without it, any number of stacked linear transformations would still be a single linear transformation — because the composition of linear maps is linear.

The Transformer Architecture — Attention as a Transformation

The most important neural architecture of the last decade, the Transformer (the "T" in GPT), uses a mechanism called self-attention. In it, each token in a sequence is transformed into three vectors — Query, Key, and Value — via three different linear transformations. The attention scores are then dot products of queries and keys, normalised by a softmax, used to weight a sum of value vectors. The entire mechanism is a sophisticated composition of linear transformations. Understanding this is the first step to understanding why ChatGPT, Gemini, and Claude work the way they do.

🏭 IoT Application
In predictive maintenance for industrial equipment, raw sensor signals (vibration, temperature, current) are transformed via filter matrices (a type of linear transformation) into feature representations that are much more informative — e.g., frequency domain features via the Fourier transform, which is itself a linear transformation encoded in a matrix called the DFT matrix.
Act Three

Eigendecomposition: The Skeleton of a Transformation

Among all the vectors in a space, there are certain special directions that a transformation simply stretches or compresses — it does not rotate them at all. These directions are called eigenvectors, and the factor by which they are stretched is the corresponding eigenvalue. Formally: if Av = λv, then v is an eigenvector of A with eigenvalue λ.

This is not a curiosity. The eigenvectors reveal the natural axes of a transformation — the bones and skeleton of what the matrix actually does. And eigendecomposition — A = QΛQᵀ — lets us write any symmetric matrix as a sum of rank-1 matrices, each scaled by its eigenvalue.

Why This Makes Repeated Transformations Cheap

Suppose you need to apply the same linear transformation A many times to a vector: Aᵏx. Without eigendecomposition, this requires k matrix multiplications, each costing O(n³) operations. With eigendecomposition:

A = QΛQ⁻¹
Aᵏ = QΛᵏQ⁻¹ ← λᵢ raised to power k; O(n) not O(n³·k)

Λᵏ = diag(λ₁ᵏ, λ₂ᵏ, ..., λₙᵏ) ← trivial to compute

Raising a diagonal matrix to a power is trivial — just raise each diagonal entry. This is why eigendecomposition matters profoundly for any algorithm that involves repeated transformations: Markov chains, PageRank, recurrent neural network dynamics, graph-based algorithms.

The Spectrum of a Matrix

The set of all eigenvalues is called the spectrum of a matrix. It carries deep information:

  • A positive definite matrix (all eigenvalues positive) corresponds to a transformation that stretches everything — used in covariance matrices, which must be positive semidefinite.
  • The largest eigenvalue governs the long-run behaviour of a linear dynamical system — whether it explodes, decays, or oscillates.
  • The condition number (ratio of largest to smallest eigenvalue) tells you how numerically ill-conditioned a system is — crucial for stable training of neural networks.

Google PageRank: Eigenvectors at Internet Scale

Google's original search algorithm, PageRank, assigns importance scores to web pages. The importance vector is the dominant eigenvector of a matrix encoding the link structure of the entire web. Billions of pages, one matrix, one eigenvector. The internet is ranked by linear algebra.

Spectral Graph Theory in IoT Networks

In an IoT network of connected sensors, the graph Laplacian matrix encodes the network topology. Its eigenvalues reveal the number of connected components, the speed of information diffusion, and the optimal clustering of sensors. Graph Neural Networks (GNNs), which operate on sensor networks, smart grids, and molecular structures, are built entirely on spectral graph theory — on eigenvalues and eigenvectors of graph matrices.

eigendecomposition_demo.py 
import numpy as np

# Symmetric covariance-like matrix
A = np.array([[4, 2, 1],
              [2, 3, 1],
              [1, 1, 2]])

eigenvalues, eigenvectors = np.linalg.eigh(A)

print("Eigenvalues (spectrum):", np.round(eigenvalues, 4))
print("Eigenvectors (columns):")
print(np.round(eigenvectors, 4))

# Reconstruct A from eigendecomposition
Q, Lambda = eigenvectors, np.diag(eigenvalues)
A_reconstructed = Q @ Lambda @ Q.T
print("\nReconstruction error:", np.round(np.linalg.norm(A - A_reconstructed), 10))

# Variance explained by each eigenvector
total = eigenvalues.sum()
variance_explained = eigenvalues / total
print("\nVariance explained (%):", np.round(variance_explained * 100, 2))
Act Four

Functions That Model Real-World Data

Before a machine can learn from data, a human must choose an appropriate mathematical model — a function class — that could plausibly have generated the data. This choice is one of the most consequential decisions in applied machine learning, and it requires understanding the qualitative behaviour of different types of functions.

A Taxonomy of Functions in AI/ML

Linear & Affine
f(x) = mx + b and f(x) = Wx + b
The building block of all neural networks. Used in Linear Regression, Logistic Regression (with sigmoid), and as layers in deep networks. Linear functions cannot model curves, but they are the substrate on which all more complex functions are built.
Polynomial
f(x) = aₙxⁿ + ... + a₁x + a₀
Used in polynomial regression to capture non-linear trends. However, high-degree polynomials overfit dramatically — this is a lesson in the bias-variance tradeoff. Kernel methods (SVMs with polynomial kernels) use them implicitly in high dimensions.
Exponential & Logarithmic
f(x) = eˣ, f(x) = ln(x)
The exponential function appears in the softmax (converting neural network outputs to probabilities), in the Gaussian distribution, and in information-theoretic losses like cross-entropy. The log is used to convert products to sums — critical for numerical stability in probabilistic models and for defining entropy.
Sigmoid & Logistic
σ(x) = 1 / (1 + e⁻ˣ)
The canonical activation function for binary classification. Its output lies in (0,1) — interpretable as a probability. Appears in logistic regression, LSTMs (as gate activations), and attention mechanisms. Its saturation at extremes is also the source of the vanishing gradient problem.
ReLU & Variants
f(x) = max(0, x)
The most commonly used activation function in modern deep learning. It is piecewise linear, computationally cheap, and avoids the vanishing gradient problem that plagued sigmoid-based networks. Variants include Leaky ReLU, GELU, and Swish — the last two used in transformers.
Sinusoidal & Fourier
f(x) = A sin(ωx + φ)
Time-series data from IoT sensors (vibration, electrical signal, ECG) often has periodic structure. Fourier decomposition represents any signal as a sum of sinusoids. In transformers, positional encodings use sinusoidal functions to inject sequence-position information.

Loss Functions: Functions That Teach Machines

Every machine learning model is trained by minimising a loss function — a function that measures the discrepancy between predictions and reality. The choice of loss function defines what the model learns to optimise:

Loss Function Formula Used For Underlying Function Type
Mean Squared Error ½ Σ(ŷ−y)² Regression Polynomial (quadratic)
Binary Cross-Entropy −[y log ŷ + (1−y) log(1−ŷ)] Classification Logarithmic
Categorical Cross-Entropy −Σ yₖ log ŷₖ Multi-class Logarithmic + Softmax
Hinge Loss max(0, 1 − y·ŷ) SVM ReLU-like piecewise linear
KL Divergence Σ p log(p/q) VAEs, language models Logarithmic + probabilistic
Act Five

Derivatives and Their Meaning in Machine Learning

The derivative is the mathematical formalisation of a simple intuition: how much does the output change when the input changes by a tiny amount? Formally, f'(x) = lim_{h→0} [f(x+h) − f(x)] / h. But in machine learning, the derivative is the engine of learning itself.

The Derivative as a Signal

When you train a neural network, you want to adjust the model's parameters so that the loss function decreases. The derivative of the loss with respect to each parameter tells you exactly this: it is the signal that tells each parameter which direction to move in. A positive derivative on parameter w means increasing w would increase the loss — so you should decrease w. A negative derivative means the opposite. The magnitude tells you by how much.

Backpropagation: The Chain Rule at Scale

The algorithm that trains virtually every neural network ever deployed — backpropagation — is the chain rule of calculus applied recursively through the network's computational graph.

If L = f(g(h(x))), then:

dL/dx = dL/df · df/dg · dg/dh · dh/dx ← multiply local gradients

Backprop does this for ALL parameters in O(forward pass cost)

This was the key insight of the 1986 backpropagation paper: computing all these partial derivatives efficiently using dynamic programming. Without the chain rule and its efficient computation, deep learning would be impossible.

Derivatives of Key ML Functions

Sigmoid

σ(x) = 1/(1+e⁻ˣ)
σ'(x) = σ(x)(1 − σ(x))

This elegant self-referential form makes the sigmoid particularly tractable. Note that at saturation (x → ±∞), σ'(x) → 0 — this causes the vanishing gradient problem.

ReLU

f(x) = max(0, x)
f'(x) = 1 if x > 0, else 0

The derivative is 1 for positive inputs and 0 for negative — a simple indicator function. This is why ReLU avoids the vanishing gradient problem: gradients don't shrink as they pass through active neurons.

Automatic Differentiation

Modern deep learning libraries (PyTorch, TensorFlow, JAX) implement automatic differentiation — a system that computes derivatives of arbitrary programs exactly, not via symbolic algebra or numerical approximation. This is built on the chain rule applied to a directed acyclic computational graph. When you call loss.backward() in PyTorch, you are triggering a traversal of this graph that computes the gradient of the loss with respect to every trainable parameter in the entire network — potentially millions of them — in a single backward pass.

💡 Interpretation
The gradient vector ∇L(w) points in the direction of steepest increase of the loss function in parameter space. To train the model, we walk in the opposite direction. This is gradient descent — the subject of Section 9. The derivative is the compass; gradient descent is the journey.
Act Six

Maxima, Minima, and the Art of Optimisation

Training a machine learning model is, at its core, an optimisation problem: find the parameter values that minimise the loss function. Every technique you learned for finding maxima and minima — setting derivatives to zero, second derivative tests, critical points — forms the conceptual foundation for the optimisation algorithms that power AI.

The Fundamental Condition

A necessary condition for a smooth function to have a local extremum at a point is that its derivative (or gradient, in the multivariable case) vanishes there. This is the critical point condition. In 1D: f'(x*) = 0. In nD: ∇f(x*) = 0.

Convex vs. Non-Convex Loss Landscapes

Linear regression has a convex loss function — a bowl-shaped parabola with exactly one minimum. Setting the derivative to zero and solving gives the closed-form solution: w* = (XᵀX)⁻¹Xᵀy. This is the Normal Equation, and it is the exact minimum, found analytically.

Deep neural networks have non-convex loss landscapes — wildly complex surfaces with millions of saddle points, local minima, and flat regions. In these landscapes, we cannot solve ∇L = 0 analytically. Instead, we use iterative numerical optimisation methods, most importantly gradient descent and its variants.

Beyond Basic Gradient Descent: Optimisation Algorithms

Algorithm Key Idea Used In
SGD Use gradient of a mini-batch instead of full dataset Classic ML, PyTorch
Momentum Accumulate velocity in gradient direction Fast convergence in smooth regions
RMSProp Adapt learning rate per-parameter based on magnitude RNNs, non-stationary objectives
Adam First + second moment estimates (momentum + RMSProp) Transformers, CNNs, default choice
L-BFGS Approximate inverse Hessian for second-order steps Small models, classical ML

All of these are refinements on the single idea of following the gradient downhill. The second-order methods (like L-BFGS) also use the Hessian matrix — the matrix of second derivatives — to take smarter, more curved steps. The Hessian's eigenvalues tell you about the curvature of the loss landscape in different directions.

📦 Supply Chain Application
Logistics companies (Amazon, DHL, Maersk) solve optimisation problems with millions of variables: delivery routes, warehouse locations, inventory levels. These are often non-convex, mixed-integer optimisation problems. The tools of calculus — gradients, curvature, constraint handling via Lagrange multipliers — underpin every solver used to make these decisions.
Act Seven

Integration: Accumulation, Probability, and Area Under the Curve

Integration is the calculus of accumulation. Where differentiation asks "how fast is this changing?", integration asks "how much has accumulated in total?" This distinction maps precisely onto two different kinds of questions in data science: what is the distribution of this variable? (answered by probability density functions and their integrals) and how well does this model perform across all operating points? (answered by metrics that integrate performance curves).

Probability Distributions as Integrals

A probability density function (PDF) p(x) is defined by two conditions: p(x) ≥ 0 for all x, and ∫ p(x) dx = 1. Every probability statement about a continuous random variable is an integral of its PDF. The expected value of a function under a distribution — the foundation of risk, loss, and Bayesian inference — is an integral:

E[f(X)] = ∫ f(x) p(x) dx

Expected loss: E[L(ŷ, y)] = ∫∫ L(ŷ, y) p(x,y) dx dy
This is what gradient descent estimates with a mini-batch!

The AUC-ROC: Integration in Model Evaluation

The Area Under the ROC Curve (AUC-ROC) is one of the most widely used metrics for binary classifiers. It is, literally, an integral — the area under the curve plotting True Positive Rate against False Positive Rate as the classification threshold varies. A model with AUC = 1.0 is perfect; AUC = 0.5 is random. The integral summarises performance across all possible operating points, not just at one chosen threshold.

Bayesian Inference: Learning via Integration

Bayesian machine learning is built entirely on integrals. Bayes' theorem:

p(θ | X) = p(X | θ) p(θ) / p(X)

p(X) = ∫ p(X | θ) p(θ) dθ ← marginal likelihood; often intractable!

The denominator p(X) requires integrating over all possible parameter values — an often infinite-dimensional integral. Much of modern Bayesian deep learning is devoted to approximating this integral efficiently (via variational inference, Markov Chain Monte Carlo, or Laplace approximations).

Integration in IoT: Signal Processing

In a motion sensor (accelerometer), the integral of acceleration gives velocity; the double integral gives position. GPS-denied navigation systems (used in drones, submarines, guided missiles) use numerical integration of inertial sensor data — accumulated over time — to estimate position. This is called Dead Reckoning, and every integral error compounds over time — which is why understanding both the power and the limitations of numerical integration matters enormously in embedded systems.

📊 Practical Python: Numerical Integration
from scipy import integrate
result, error = integrate.quad(lambda x: x**2 * np.exp(-x**2), 0, np.inf)

SciPy's quad function uses adaptive Gaussian quadrature — a sophisticated numerical integration technique that estimates error and refines the approximation adaptively. This is used in everything from option pricing (Black-Scholes model) to computing normalisation constants in probabilistic models.
Act Eight

Multivariable Functions: High-Dimensional Landscapes

Real-world ML models do not have one or two parameters. They have millions — sometimes hundreds of billions. The loss function is a function of all these parameters simultaneously: L: ℝᵈ → ℝ, a scalar-valued function of a high-dimensional vector. Multivariable calculus is the mathematics of navigating this landscape.

Visualising High-Dimensional Functions

While we cannot directly visualise functions in millions of dimensions, several techniques provide insight:

Contour Plots

For 2D parameter spaces, contour plots show level sets — curves along which the function has equal value. The gradient is always perpendicular to contour lines, pointing toward higher values. Watching a gradient descent trajectory on a contour plot is one of the most illuminating ways to understand the algorithm.

Loss Landscape Visualisation

Researchers visualise neural network loss landscapes by picking two random directions in parameter space and plotting the loss over a 2D grid. These visualisations reveal the dramatic difference between flat, well-conditioned landscapes (ResNets with skip connections) and rough, chaotic ones (plain deep networks).

Heatmaps of Attention

In transformers, attention scores form a matrix (a 2D function of token positions). Visualising this heatmap shows which tokens attend to which — a direct window into what the model is "thinking about."

t-SNE and UMAP Embeddings

These algorithms project high-dimensional data (e.g., the 768-dimensional embeddings of BERT) into 2D for visualisation, preserving local neighbourhood structure. They are non-linear dimensionality reduction tools built on multivariable optimisation.

Partial Derivatives: Taking One Dimension at a Time

The partial derivative ∂f/∂xᵢ measures how the function changes as we vary the i-th input while holding all others constant. This is exactly what the backpropagation algorithm computes: the partial derivative of the loss with respect to each individual weight, one at a time — while the chain rule efficiently couples all these computations together.

🌡️ IoT: Multivariable Physical Models
In a smart building energy management system, the thermal comfort of a room is a function of temperature, humidity, air velocity, and radiant temperature — a 4+ variable function. The partial derivative with respect to each variable tells the HVAC system how sensitive comfort is to each control action, enabling optimal multi-variable control. This is Model Predictive Control (MPC), one of the most important control frameworks in industrial IoT.
Act Nine

Gradient Descent: The Algorithm That Trains AI

We have arrived at one of the two capstone algorithms — the one that powers learning itself. Gradient descent is disarmingly simple in concept and extraordinarily powerful in practice. It is the reason every major AI model in the world was able to be trained.

The Algorithm

Given a loss function L(w) and a starting point w₀, gradient descent iteratively updates:

w_{t+1} = wₜ − η · ∇L(wₜ)

η = learning rate (step size hyperparameter)
∇L = gradient vector (computed by backprop)
The update subtracts a scaled gradient → moves downhill

The Mathematics Behind the Update

Why does moving in the negative gradient direction decrease the loss? This follows directly from the first-order Taylor expansion of L around the current point:

L(w + δ) ≈ L(w) + ∇L(w)ᵀ δ + O(‖δ‖²)

Set δ = −η ∇L(w):
L(w − η∇L) ≈ L(w) − η ‖∇L(w)‖²

≤ L(w) ← the loss decreases! (for small enough η)

The key insight: the squared norm of the gradient is always non-negative, so subtracting η ‖∇L‖² from the loss is guaranteed to decrease it (as long as the learning rate is small enough and we haven't reached a critical point). This is the mathematical guarantee behind gradient descent.

The Learning Rate: The Most Important Hyperparameter

The learning rate η controls the step size. Too large, and the updates overshoot — the algorithm oscillates or diverges. Too small, and convergence is impossibly slow. Finding the right learning rate is one of the most consequential practical challenges in training deep learning models.

  • Learning Rate Schedules: Warm-up phases (used in transformers) start with a small learning rate that ramps up before decaying. Cosine annealing reduces the learning rate following a cosine curve.
  • Adaptive Methods: Adam and RMSProp adapt the learning rate per-parameter based on gradient history, largely solving this problem for practitioners.
  • Learning Rate Finders: A technique that tests many learning rates in a short run to find the optimal regime before full training begins.
gradient_descent_from_scratch.py 
import numpy as np
import matplotlib.pyplot as plt

# 2D loss landscape: L(w1, w2) = w1² + 5·w2²  (elliptical bowl)
def loss(w):
    return w[0]**2 + 5 * w[1]**2

def gradient(w):
    return np.array([2 * w[0], 10 * w[1]])

# Run gradient descent
w = np.array([4.0, 3.0])   # starting point
eta = 0.08                 # learning rate
path = [w.copy()]

for step in range(40):
    grad = gradient(w)
    w = w - eta * grad    # gradient descent update
    path.append(w.copy())
    if np.linalg.norm(grad) < 1e-6:
        print(f"Converged at step {step}")
        break

path = np.array(path)
print(f"Final w: {np.round(w, 6)}, Loss: {loss(w):.8f}")

# Visualise trajectory on contour plot
w1s = np.linspace(-5, 5, 300)
w2s = np.linspace(-4, 4, 300)
W1, W2 = np.meshgrid(w1s, w2s)
Z = W1**2 + 5 * W2**2

plt.figure(figsize=(8, 6))
plt.contourf(W1, W2, Z, levels=30, cmap='YlOrRd')
plt.contour(W1, W2, Z, levels=15, colors='white', linewidths=0.5)
plt.plot(path[:,0], path[:,1], 'b.-', markersize=5, label='GD path')
plt.scatter(*path[0], color='blue', s=80, zorder=5, label='Start')
plt.scatter(*path[-1], color='green', s=80, zorder=5, label='End')
plt.legend()
plt.title('Gradient Descent Trajectory')
plt.savefig('gd_trajectory.png', dpi=150, bbox_inches='tight')

From Single Weights to Billion-Parameter Models

The exact same update rule above — w ← w − η∇L(w) — is applied to the billions of parameters in GPT-4, LLaMA, Gemini, and every other large language model in existence. The mathematical principle is identical to what you just computed. The engineering challenge is doing it efficiently across thousands of GPUs, with batches of data, with careful numerics — but the mathematical heart is unchanged. When you understood gradient descent, you understood how intelligence is learned from data.

Act Ten

Principal Component Analysis: Linear Algebra Meets Statistics

Principal Component Analysis (PCA) is the capstone of the linear algebra arc — the algorithm where vectors, matrices, norms, linear transformations, and eigendecomposition all converge into a single, beautiful, deeply useful method. It is one of the most important algorithms in all of data science, and understanding it fully requires everything you have learned.

The Problem PCA Solves

High-dimensional data is hard to visualise, expensive to store, and contains redundancy (correlated features). PCA finds a new coordinate system — one where the axes are aligned with the directions of maximum variance in the data. These new axes are the principal components, and we can project the data onto just the first few of them without losing much information.

The Algorithm, Step by Step

Step 1Centre the data: subtract the mean of each feature
Step 2Compute the covariance matrix C = XᵀX / (n−1)
Step 3Eigendecompose C = QΛQᵀ
Step 4Sort eigenvectors by eigenvalue (descending)
Step 5Project: Z = XQ_k (use top k eigenvectors)

Why Eigenvectors of the Covariance Matrix?

The covariance matrix C = XᵀX/(n−1) encodes how features vary together. Its eigenvectors are exactly the directions in feature space where variance is maximised (or minimised). The corresponding eigenvalue is the variance in that direction. So the first principal component (eigenvector with the largest eigenvalue) is the single direction that captures the most variance in the dataset.

Find unit vector v that maximises the projected variance:

v* = argmax_v vᵀCv subject to ‖v‖ = 1

By Lagrange multipliers, this is equivalent to:
Cv = λv ← eigenvalue equation!

This is the elegant mathematical moment: the optimisation problem for finding the direction of maximum variance is exactly the eigenvalue problem. The two arcs — linear algebra and calculus/optimisation — meet in a single equation.

Singular Value Decomposition (SVD): The Generalisation

In practice, PCA is computed via SVD rather than eigendecomposition of the covariance matrix. SVD decomposes any matrix X = UΣVᵀ, where U and V are orthogonal and Σ is diagonal with non-negative entries (singular values). The columns of V are the principal components; the singular values squared are proportional to eigenvalues of the covariance matrix. SVD is numerically more stable and works directly on the data matrix, not the covariance matrix.

pca_from_scratch.py 
import numpy as np
from sklearn.datasets import load_digits
import matplotlib.pyplot as plt

# Load the digits dataset: 1797 images, 64 features (8×8 pixels)
digits = load_digits()
X, y = digits.data, digits.target  # X: (1797, 64)

# ── PCA from scratch ──────────────────────────────────
# Step 1: Centre the data
X_c = X - X.mean(axis=0)

# Step 2: Compute covariance matrix (64×64)
C = (X_c.T @ X_c) / (X_c.shape[0] - 1)

# Step 3: Eigendecompose
eigenvalues, eigenvectors = np.linalg.eigh(C)

# Step 4: Sort descending
idx = np.argsort(eigenvalues)[::-1]
eigenvalues, eigenvectors = eigenvalues[idx], eigenvectors[:, idx]

# Variance explained
var_exp = eigenvalues / eigenvalues.sum()
cum_var = np.cumsum(var_exp)
print(f"Components for 90% variance: {(cum_var >= 0.90).argmax() + 1}")

# Step 5: Project onto first 2 PCs for visualisation
Z = X_c @ eigenvectors[:, :2]  # (1797, 2)

plt.figure(figsize=(9, 7))
scatter = plt.scatter(Z[:, 0], Z[:, 1], c=y, cmap='tab10', 
                      alpha=0.6, s=15)
plt.colorbar(scatter, label='Digit Class')
plt.title('PCA of Digits Dataset (64D → 2D)')
plt.xlabel('First Principal Component')
plt.ylabel('Second Principal Component')
plt.savefig('pca_digits.png', dpi=150, bbox_inches='tight')

PCA in the Real World

🧬

Genomics

Human genome studies measure millions of SNPs per person. PCA reduces these to a handful of principal components that capture ancestry — enabling population structure analysis and removing confounders from genome-wide association studies.

📸

Facial Recognition (Eigenfaces)

The original Eigenfaces algorithm (1991) applied PCA to a dataset of face images. Each face is a vector in pixel space; the principal components are "eigenfaces." New faces are projected onto this basis for recognition — a direct application of eigendecomposition to vision.

📊

Finance — Factor Models

In quantitative finance, the returns of hundreds of stocks are explained by a small number of latent factors (market, sector, momentum). PCA on the returns covariance matrix extracts these factors — this is the basis of Statistical Arbitrage strategies.

🏭

Industrial IoT — Anomaly Detection

Manufacturing equipment with 200+ sensors: PCA reduces these to 10–15 principal components. Anomalies appear as points far from the PCA subspace (high reconstruction error). This is widely used for predictive maintenance in semiconductor fabs, power plants, and automotive manufacturing.

The Intersection of Your Topics with Emerging Fields

🧠

Large Language Models

Transformers are chains of linear transformations (attention + feedforward layers), trained via gradient descent with backpropagation, operating on token embedding vectors. Every concept you have learned is inside GPT.

🤖

Reinforcement Learning

The value function and policy are multivariable functions. Policy gradient methods use the gradient of the expected reward. Bellman equations are linear operators on value functions — eigenvalues determine convergence speed.

🌐

Graph Neural Networks

GNNs operate on the graph Laplacian — a matrix whose eigenspectrum encodes the graph's structure. Message passing in GNNs is a sequence of linear transformations on node feature vectors. Used in molecular design, traffic prediction, and social networks.

🎨

Generative AI (Diffusion)

Stable Diffusion and DALL-E use score matching — computing the gradient of a log-probability function with respect to input pixels. The denoising process is guided by this score (a gradient), iteratively refined. Generating an image is literally gradient descent in pixel space.

🏙️

Edge AI & IoT

Deploying models on edge devices (microcontrollers, FPGAs) requires model compression. Techniques like weight pruning (setting small-norm weights to zero) and quantisation-aware training use linear algebraic analysis of weight matrices. PCA-based compression reduces model size for embedded deployment.

🔐

Privacy-Preserving ML

Federated Learning aggregates gradients from edge devices without centralising data. Differential Privacy adds calibrated noise to gradients — the noise magnitude is set based on the sensitivity (a norm of the gradient). Both rely on vector calculus and linear algebra.
